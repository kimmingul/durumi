# 편집 모드 가이드 (v0.1.11~)

Durumi의 편집 화면은 **3가지 모드** 중 하나로 작동합니다. 작성 중인
문서는 동일한 markdown 소스이지만, **어떻게 보여줄지**만 모드에 따라
바뀝니다. 모드는 언제든 즉시 전환 가능 — 문서 내용이나 커서/스크롤
위치는 보존됩니다.

> **v0.1.13에서 모드 명칭 변경**: 사용자 친화적인 이름으로 정비.
> "WYSIWYG" → **문서 (Document)**,
> "Typora 스타일" → **라이브 (Live)**,
> "Markdown 소스" → **소스 (Source)**.
> 내부 prefs key (`prefs.editor.defaultMode`)는 호환성을 위해 기존
> `wysiwyg | typora | markdown` 값을 그대로 유지.

---

## 모드 비교

| 모드 | 마크다운 마커 | 툴바 | 사용 시기 |
|:--|:--|:--|:--|
| **문서** *(기본, Document)* | 화면에 **표시 안 함** (모든 줄, 활성 줄 포함) | 상단에 서식 툴바 | MS Word에서 작업하던 느낌 그대로. 마우스/툴바로 모든 서식 적용. 마크다운을 의식하지 않고 글에 집중 |
| **라이브** *(Live)* | 비활성 줄에서만 숨김 / 활성 줄은 raw | 없음 | v0.1.0~v0.1.10의 라이브 프리뷰 동작. 키보드로 마크다운을 직접 치고 싶을 때. 마크다운 사용자에게 친숙 |
| **소스** *(Source)* | 항상 raw, 라이브 프리뷰 OFF | 없음 | 마크다운 문법 자체를 다루거나 텍스트 에디터처럼 쓰고 싶을 때 |

## 모드 전환

3가지 방법:

1. **상태바 우측 segmented control** (D / L / S 아이콘) — 항상 보이며
   현재 모드가 강조됨. 한 번 클릭으로 즉시 전환.
2. **보기 → 편집 모드** 라디오 메뉴
3. **단축키**:
   - `Cmd/Ctrl + Shift + 1` — 문서 (Document)
   - `Cmd/Ctrl + Shift + 2` — 라이브 (Live)
   - `Cmd/Ctrl + Shift + 3` — 소스 (Source)
   - `Cmd/Ctrl + /` — 소스 ↔ 이전 모드 토글

선택한 모드는 `prefs.editor.defaultMode`로 자동 저장되어 다음 실행
시에도 유지됩니다.

## 문서 모드 자세히

### 동작 원리 (v0.1.12부터 강화)

문서 모드에서는 **타이핑한 마크다운 기호를 자동으로 이스케이프**합니다.

| 사용자 타이핑 | 저장된 소스 | 화면 표시 |
|:--|:--|:--|
| `#` | `\#` | `#` (글자) |
| `# Heading` | `\# Heading` | `# Heading` (글자, **heading 아님**) |
| `*foo*` | `\*foo\*` | `*foo*` (글자, **italic 아님**) |
| `> quote` | `\> quote` | `> quote` (글자, blockquote 아님) |
| `1. item` (라인 시작) | `1\. item` | `1. item` (글자, 번호 리스트 아님) |
| `[foo]` | `\[foo\]` | `[foo]` (글자, link 아님) |
| `<sup>1</sup>` | `\<sup\>1\</sup\>` | `<sup>1</sup>` (글자, HTML 위첨자 아님) |

서식을 적용하려면 **반드시 툴바 또는 단축키**를 사용:
- 굵게: Bold 버튼 또는 `Cmd+B`
- Heading 1: Style 드롭다운 → "제목 1" 또는 `Cmd+1`
- 인용: Insert citation 버튼 (cite palette)
- 등등

### 기존 마크다운 문서

이미 작성된 마크다운 (`# Hello` 등)을 열면 **그대로 heading으로
렌더링**됩니다. 자동 이스케이프는 *새로 타이핑하는 문자*에만
적용됩니다. Style 드롭다운 → "본문"으로 heading을 일반 텍스트로
변환 가능.

### 붙여넣기

붙여넣기 내용은 escape **안 합니다**. 다른 마크다운 문서를 복사해
오면 서식이 보존됩니다.

### IME 안전성

v0.1.12부터 invariant #1이 완화되었습니다: **content widget의
`Decoration.replace`만 활성 줄 금지** (image, math, mermaid, table 등).
Inline 마커 hiding (empty `cm-md-marker-hidden` widget으로 `**`, `[`,
`<`, `#` 등 punctuation을 숨기는 것)은 활성 줄에서도 안전 — 사용자가
punctuation에 한국어 합성을 시도하지 않으므로 IME composition 영향
없음.

이 완화 덕분에 문서 모드는 **활성 라인과 비활성 라인을 동일하게**
렌더링할 수 있고, 각 decoration plugin이 자기 마커만 책임지면 됩니다
(별도 패치 hider 불필요). 이스케이프 필터도 IME composition 트랜잭션
(`input.compose`)을 건드리지 않습니다.

### autoPair

문서 모드에서 인라인 마커 (`*`, `_`, `` ` ``, `[`, `~`, `=`)의
**autoPair는 비활성화**됩니다 (escape와 충돌 방지). 일반 괄호 (`(`,
`{`, `<`) 와 따옴표는 정상 동작.

### 블록 위젯 (이미지/수식/Mermaid/표/HR/각주/인용 등)

**v0.1.12부터 변경**: 문서 모드에서는 **모든 widget이 활성 라인에도
렌더**됩니다. 이미지, 표, 수식(인라인+블록), Mermaid, HR, 체크박스,
각주 pill, 인용 pill, frontMatter 요약 — 라인 위로 캐럿이 와도 위젯
그대로 표시되어 Word와 같은 일관성을 유지합니다.

라이브 모드에서는 v0.1.0~v0.1.11과 동일하게 활성 라인에서 원본 source가
보입니다 (편집 affordance).

캐럿이 위젯 영역에 있을 때 한글 IME 합성이 깨지는 경우가 발견되면
알려주세요 — 해당 widget만 보수적으로 되돌리거나 mark+display:none+
side-widget 패턴으로 마이그레이션 가능합니다. (현재 CodeMirror 6의
composition 경계 자동 bailout을 신뢰)

### 툴바 (문서 모드에서만)

| 그룹 | 버튼 |
|:--|:--|
| **Style** | Body / 제목 1~6 / 인용문 / 코드 블록 (드롭다운) |
| **Inline** | 굵게 / 기울임 / 취소선 / 인라인 코드 / 위 첨자(`<sup>`) / 아래 첨자(`<sub>`) |
| **List** | 글머리 기호 / 번호 매기기 / 체크리스트 / 들여쓰기 / 내어쓰기 |
| **Insert** | 링크 / 이미지 / 표 / 수식 / 각주 / 참고문헌 |
| **Review** | CriticMarkup 강조 / 메모 / 변경 추적 |

스타일 드롭다운은 캐럿 위치의 줄 종류를 자동 감지해 표시값을
업데이트합니다.

## 라이브 모드 자세히

v0.1.0부터 v0.1.10까지 Durumi의 기본이었던 모드입니다. 사용자가
편집 중인 줄(활성 줄)에서만 마크다운 markers를 raw 형태로 보여주고,
다른 줄들은 렌더된 결과만 표시합니다.

직접 markdown을 타이핑하는 워크플로(`**`, `_`, `#` 등을 키보드로
입력)에 익숙한 사용자에게 적합합니다. 문서 모드에서는 마커가 숨겨져
있어 키 시퀀스가 자연스럽지 않을 수 있습니다.

---

## 소스 모드 자세히

모든 라이브 프리뷰 데코레이션이 OFF. 평문 markdown 소스만 표시되고
syntax highlighting (Lezer)만 적용됩니다. 일반 텍스트 에디터처럼 동작.

| 기능 | 동작 |
|:--|:--|
| `**bold**` | 그대로 표시 |
| 이미지 | URL/경로만 표시 (위젯 렌더 안 함) |
| 수식 | `$$...$$` 원본 |
| 표 | markdown 파이프 표 |

뭔가가 이상해 보일 때 raw 소스를 빠르게 확인하는 디버그 도구로도
유용합니다.

## 문서 스타일 (Settings → 문서 스타일)

문서·라이브 모드에서 보이는 글꼴 / 크기 / 줄간격을 사용자 선호에
맞게 조정할 수 있습니다.

### 6개 사전 제공 프리셋

| 프리셋 | 본문 | 용도 |
|:--|:--|:--|
| Durumi 기본 | Inter 16px, 1.6 | 기본값, 일상 작성 |
| 전형적 매뉴스크립트 | Times New Roman 12pt, 2.0 (double) | NEJM/JAMA 투고 분량 가늠 |
| Nature 스타일 | Helvetica 14px, 1.5 | 분자/세포생물 |
| Lancet 스타일 | Georgia 14px, 1.55 | 임상의학 |
| JKMS / 국내 의학지 | Noto Serif KR 16px, 1.7 | 한국 학회지 |
| 장시간 작성 친화 | Atkinson Hyperlegible 17px, 1.75 | 1차 초안 집필 |

> **주의**: 프리셋 이름(예: "Nature 스타일")은 **화면 표시용** 드래프트
> 스타일입니다. 실제 저널 조판 템플릿이 아닙니다. 최종 투고는 기존
> export 파이프라인(`csl: <style>.csl`, Pandoc reference-doc 등)으로
> 진행하세요.

### 항목별 미세 조정

10개의 스타일 항목 (Body / H1-H6 / Blockquote / Code / Table header)
각각에 대해:

- Font family (시스템 폰트 이름 자유 입력)
- Size (px)
- Weight (400 / 600 / 700)
- Color (선택 안 함 = 테마 기본값)
- Line height (배수, 예: 1.5)

항목 하나라도 수정하면 프리셋 드롭다운이 "Custom (사용자 지정)"으로
바뀝니다.

### **Durumi 기본으로 되돌리기**

Settings 탭 우측 상단의 **[초기화]** 버튼 — 한 번에 모든 스타일을
Durumi 기본 프리셋으로 복원.

### 내보내기 연동

스타일은 50개의 CSS 변수(`--style-body-font`, `--style-h1-size`,
…)로 `:root`에 inject됩니다. HTML/PDF 내보내기 파이프라인이 같은
변수를 사용하므로 **화면에서 본 글꼴과 출력 결과가 일치**합니다.
단, PDF 출력은 해당 폰트가 시스템에 설치되어 있어야 합니다.

## 자주 묻는 질문

**Q. 모드를 바꿔도 문서 내용이 바뀌나요?**
A. 아니요. 모드는 *표시 방식*만 결정합니다. 저장되는 markdown 소스는
   동일합니다.

**Q. 문서 모드에서 표를 편집하려면?**
A. 표 셀에 캐럿을 두면 자동으로 raw 소스(`| col1 | col2 |`)가 보입니다.
   다른 줄로 이동하면 다시 렌더된 표가 보입니다.

**Q. 문서 모드에서 마크다운 단축키(`**`로 굵게 등)가 동작하나요?**
A. v0.1.12부터 **변경**: 타이핑한 `**`은 자동으로 `\*\*`로 escape되어
   글자로 인식됩니다. 굵게는 **`Cmd+B`** 또는 툴바 Bold 버튼을 쓰세요.
   같은 결과를 더 안전하게 얻습니다.

**Q. 문서 모드에서 `[@cite]` autocomplete가 작동하나요?**
A. v0.1.12부터 **작동 안 함**. `[`이 escape되어 `\[`이 되므로
   autocomplete가 발동하지 않습니다. 인용은 **툴바의 Citation 버튼**
   또는 `Cmd+Shift+I`로 cite palette를 열어 삽입하세요. 라이브 /
   소스 모드에서는 `[@` autocomplete가 그대로 동작합니다.

**Q. 소스 모드에서 작성한 내용을 문서 모드에서 보면 다르게 보이나요?**
A. 같은 문서지만 **렌더된 모습**으로 표시됩니다. 마크다운 문법은
   숨겨지고 결과만 보입니다.

**Q. 문서 모드에서 한국어 IME가 잘 동작하나요?**
A. 네. 마커 hide는 `Decoration.mark`(텍스트 노드 보존)만 사용하므로
   IME composition 영역과 충돌하지 않습니다.

**Q. 스타일 프리셋을 저장하면 다른 컴퓨터에서도 보이나요?**
A. 글로벌 prefs로 저장됩니다 (`~/Library/Application Support/Durumi/preferences.json`
   on macOS). 다른 컴퓨터로 옮기려면 이 파일을 복사하세요.

# 편집 모드 가이드 (v0.1.11~)

Durumi의 편집 화면은 **3가지 모드** 중 하나로 작동합니다. 작성 중인
문서는 동일한 markdown 소스이지만, **어떻게 보여줄지**만 모드에 따라
바뀝니다. 모드는 언제든 즉시 전환 가능 — 문서 내용이나 커서/스크롤
위치는 보존됩니다.

---

## 모드 비교

| 모드 | 마크다운 마커 | 툴바 | 사용 시기 |
|:--|:--|:--|:--|
| **WYSIWYG** *(기본)* | 화면에 **표시 안 함** (모든 줄, 활성 줄 포함) | 상단에 서식 툴바 | Word에서 작업하던 느낌 그대로. 마우스/툴바로 모든 서식 적용 |
| **Typora 스타일** | 비활성 줄에서만 숨김 / 활성 줄은 raw | 없음 | v0.1.0~v0.1.10의 동작. 키보드로 마크다운을 직접 치고 싶을 때 |
| **Markdown 소스** | 항상 raw, 라이브 프리뷰 OFF | 없음 | 마크다운 문법 자체를 다루거나 텍스트 에디터처럼 쓰고 싶을 때 |

## 모드 전환

3가지 방법:

1. **상태바 우측 segmented control** (W / T / M 아이콘) — 항상 보이며
   현재 모드가 강조됨. 한 번 클릭으로 즉시 전환.
2. **View → 편집 모드** 라디오 메뉴
3. **단축키**:
   - `Cmd/Ctrl + Shift + 1` — WYSIWYG
   - `Cmd/Ctrl + Shift + 2` — Typora 스타일
   - `Cmd/Ctrl + Shift + 3` — Markdown 소스
   - `Cmd/Ctrl + /` — Markdown ↔ 이전 모드 토글

선택한 모드는 `prefs.editor.defaultMode`로 자동 저장되어 다음 실행
시에도 유지됩니다.

## WYSIWYG 모드 자세히

### 동작 원리

마크다운 마커(`**`, `_`, `#`, `>`, 목록 표시 등)를 `Decoration.mark`로
hidden 처리하고 CSS `display: none`으로 화면에서 제거합니다. 텍스트
자체는 문서에 그대로 존재 — 커서 이동, 검색, copy/paste 모두 정상.

### IME 안전성

한국어/일본어/중국어 IME 합성을 보호하기 위해 마커 hide에 절대
`Decoration.replace`를 쓰지 않습니다. 활성 줄 invariant가 그대로
지켜집니다.

### 블록 위젯 (이미지/수식/Mermaid)

이미지·수식·Mermaid 같은 **블록 위젯**은 비활성 줄에서는 렌더링되고,
**활성 줄에서는 원본 markdown source**를 보여줍니다. 이는 의도된
편집 affordance입니다 — 활성 줄에서 원본을 보면서 수정할 수 있도록.
(클릭하면 자동으로 source가 보임)

### 툴바 (WYSIWYG 모드에서만)

| 그룹 | 버튼 |
|:--|:--|
| **Style** | Body / 제목 1~6 / 인용문 / 코드 블록 (드롭다운) |
| **Inline** | 굵게 / 기울임 / 취소선 / 인라인 코드 / 위 첨자(`<sup>`) / 아래 첨자(`<sub>`) |
| **List** | 글머리 기호 / 번호 매기기 / 체크리스트 / 들여쓰기 / 내어쓰기 |
| **Insert** | 링크 / 이미지 / 표 / 수식 / 각주 / 참고문헌 |
| **Review** | CriticMarkup 강조 / 메모 / 변경 추적 |

스타일 드롭다운은 캐럿 위치의 줄 종류를 자동 감지해 표시값을
업데이트합니다.

## Typora 모드 자세히

v0.1.0부터 v0.1.10까지 Durumi의 기본이었던 모드입니다. 사용자가
편집 중인 줄(활성 줄)에서만 마크다운 markers를 raw 형태로 보여주고,
다른 줄들은 렌더된 결과만 표시합니다.

직접 markdown을 타이핑하는 워크플로(`**`, `_`, `#` 등을 키보드로
입력)에 익숙한 사용자에게 적합합니다. WYSIWYG에서는 마커가 숨겨져
있어 키 시퀀스가 자연스럽지 않을 수 있습니다.

---

## Markdown 소스 모드 자세히

모든 라이브 프리뷰 데코레이션이 OFF. 평문 markdown 소스만 표시되고
syntax highlighting (Lezer)만 적용됩니다. 일반 텍스트 에디터처럼 동작.

| 기능 | 동작 |
|:--|:--|
| `**bold**` | 그대로 표시 |
| 이미지 | URL/경로만 표시 (위젯 렌더 안 함) |
| 수식 | `$$...$$` 원본 |
| 표 | markdown 파이프 표 |

뭔가가 이상해 보일 때 raw 소스를 빠르게 확인하는 디버그 도구로도
유용합니다.

## 문서 스타일 (Settings → 문서 스타일)

WYSIWYG/Typora 모드에서 보이는 글꼴 / 크기 / 줄간격을 사용자 선호에
맞게 조정할 수 있습니다.

### 6개 사전 제공 프리셋

| 프리셋 | 본문 | 용도 |
|:--|:--|:--|
| Durumi 기본 | Inter 16px, 1.6 | 기본값, 일상 작성 |
| 전형적 매뉴스크립트 | Times New Roman 12pt, 2.0 (double) | NEJM/JAMA 투고 분량 가늠 |
| Nature 스타일 | Helvetica 14px, 1.5 | 분자/세포생물 |
| Lancet 스타일 | Georgia 14px, 1.55 | 임상의학 |
| JKMS / 국내 의학지 | Noto Serif KR 16px, 1.7 | 한국 학회지 |
| 장시간 작성 친화 | Atkinson Hyperlegible 17px, 1.75 | 1차 초안 집필 |

> **주의**: 프리셋 이름(예: "Nature 스타일")은 **화면 표시용** 드래프트
> 스타일입니다. 실제 저널 조판 템플릿이 아닙니다. 최종 투고는 기존
> export 파이프라인(`csl: <style>.csl`, Pandoc reference-doc 등)으로
> 진행하세요.

### 항목별 미세 조정

10개의 스타일 항목 (Body / H1-H6 / Blockquote / Code / Table header)
각각에 대해:

- Font family (시스템 폰트 이름 자유 입력)
- Size (px)
- Weight (400 / 600 / 700)
- Color (선택 안 함 = 테마 기본값)
- Line height (배수, 예: 1.5)

항목 하나라도 수정하면 프리셋 드롭다운이 "Custom (사용자 지정)"으로
바뀝니다.

### **Durumi 기본으로 되돌리기**

Settings 탭 우측 상단의 **[초기화]** 버튼 — 한 번에 모든 스타일을
Durumi 기본 프리셋으로 복원.

### 내보내기 연동

스타일은 50개의 CSS 변수(`--style-body-font`, `--style-h1-size`,
…)로 `:root`에 inject됩니다. HTML/PDF 내보내기 파이프라인이 같은
변수를 사용하므로 **화면에서 본 글꼴과 출력 결과가 일치**합니다.
단, PDF 출력은 해당 폰트가 시스템에 설치되어 있어야 합니다.

## 자주 묻는 질문

**Q. 모드를 바꿔도 문서 내용이 바뀌나요?**
A. 아니요. 모드는 *표시 방식*만 결정합니다. 저장되는 markdown 소스는
   동일합니다.

**Q. WYSIWYG에서 표를 편집하려면?**
A. 표 셀에 캐럿을 두면 자동으로 raw 소스(`| col1 | col2 |`)가 보입니다.
   다른 줄로 이동하면 다시 렌더된 표가 보입니다.

**Q. WYSIWYG에서 마크다운 단축키(`**`로 굵게 등)가 동작하나요?**
A. 네. CodeMirror의 기존 keymap이 그대로 적용됩니다. `**foo**`를
   타이핑하면 마커가 즉시 숨겨지고 굵은 텍스트만 보입니다.

**Q. Markdown 모드에서 작성한 내용을 WYSIWYG에서 보면 다르게 보이나요?**
A. 같은 문서지만 **렌더된 모습**으로 표시됩니다. 마크다운 문법은
   숨겨지고 결과만 보입니다.

**Q. WYSIWYG에서 한국어 IME가 잘 동작하나요?**
A. 네. 마커 hide는 `Decoration.mark`(텍스트 노드 보존)만 사용하므로
   IME composition 영역과 충돌하지 않습니다.

**Q. 스타일 프리셋을 저장하면 다른 컴퓨터에서도 보이나요?**
A. 글로벌 prefs로 저장됩니다 (`~/Library/Application Support/Durumi/preferences.json`
   on macOS). 다른 컴퓨터로 옮기려면 이 파일을 복사하세요.

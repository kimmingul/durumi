## 3. Investigation framework

The audit should produce a **feature × mode × status matrix**, like:

| Feature | Document | Live | Source | Notes |
|:--|:--:|:--:|:--:|:--|
| Heading H1–H6 | ✅ | ✅ | ✅ | dropdown in toolbar |
| Inline bold/italic/strike | ✅ | ✅ | ✅ | with active-state |
| Table cell edit | ✅ | ⚠️ | ✅ | live mode behaviour? |
| Mermaid block | ⚠️ | ⚠️ | ✅ | hover edit? |
| ... | ... | ... | ... | ... |

Status legend:
- ✅ — works correctly in this mode + has test coverage
- ⚠️ — partial / has a known gap / no test coverage
- ❌ — missing / broken
- 🚫 — not applicable in this mode (e.g. live-preview decorations in Source mode)

### Feature categories to cover

**A. Core markdown / GFM**:
- Headings (H1–H6), paragraph, line break
- Bold / italic / strikethrough / inline code / sup / sub
- Bulleted / numbered / task lists with nesting
- Blockquote
- Horizontal rule
- Fenced code block (with language tag) + indented code block
- Inline link / image / autolink
- Footnote (reference + definition)
- Front matter (YAML)
- TOC marker `[toc]`
- Table (header + body + alignment)

**B. Durumi-specific extensions**:
- CriticMarkup — all 5 kinds (insert / delete / substitute / highlight / comment)
- Inline `%%memo%%` (distinct from CriticMarkup comment)
- Citation `[@key]` / `[-@key]` / `[@a; @b]`
- Mermaid fenced block (lazy KaTeX-style render?)
- Math — inline `$x$` and block `$$ … $$`

**C. Editor operations**:
- Paste — plain text, formatted text, HTML, image, markdown
- Drag-drop — image, file, URL
- Undo / redo across mode switches
- Find / replace (per-mode behaviour)
- Multi-cursor / column selection
- Native macOS spell-check + autocorrect interaction

**D. Document operations**:
- Save (atomic, verified in v0.2.0)
- Save As / discard prompt
- Export — HTML / PDF / Pandoc DOCX / Pandoc LaTeX
- Import — Pandoc DOCX / ODT / RTF
- Mode switching — caret preserved? scroll preserved? selection?

**E. Workspace + sidebar**:
- Open folder / close folder / recent folders
- File tree — drag-drop, rename, create, trash, duplicate, reveal
- Search (in workspace) — case / regex / replace
- Outline panel
- Memo panel
- References panel (BibTeX / RIS, DOI / PubMed / Crossref / KoreaMed search)
- AI panel (palette, ghost text, citation suggest, usage)

**F. Per-mode UX**:
- Toolbar behaviour: visible only in Document mode currently. Should it appear in Live? Source? With which subset?
- Status bar / mode indicator
- Keyboard shortcuts — do all Cmd+B, Cmd+I, etc. work identically across all 3 modes?
- Caret behaviour on mode switch
- Scroll preservation on mode switch
- Find / replace UI consistency

**G. Accessibility**:
- Keyboard-only navigation through editor + sidebars + dialogs
- Screen reader ARIA labels on all controls
- Focus rings + visible focus state
- High-contrast theme support

**H. Print / export consistency**:
- The same document exported from Document mode vs Source mode — identical output?
- CriticMarkup accept vs preserve behaviour
- Image inclusion (durumi-asset:// must be inlined as data URLs in HTML export?)

## 4. Suggested session B agent plan

### Phase A — Inventory (single agent, parallelisable later)

Spawn one agent with a copy of this document. The agent's job:

1. Read the codebase systematically — every decoration in `src/editor/decorations/`, every command in `src/editor/keymap/`, every menu item in `electron/menu.ts`, every store action in `src/store/`.
2. Cross-reference each feature against the matrix in §3.
3. For each feature, exercise it manually-ish — either via existing tests, by reading the code, or by suggesting an e2e to write.
4. Produce a fully-filled matrix in markdown.
5. List the top 20 gaps ranked by user value × implementation cost.

**Output**: `docs/v0.2-feature-matrix.md` (committed). Time budget: ~60 min.

### Phase B — User negotiation

The user reviews the matrix + top 20 gaps. Picks which subset to ship
in v0.2.8 / v0.2.9 / v0.2.10. Negotiation happens in chat, no agent.

### Phase C — Per-gap implementation

For each agreed gap, spawn a dedicated implementation agent — same
pattern as the v0.2.4 / v0.2.5 / v0.2.6 / v0.2.7 cycle. Each agent:
- Implements the gap
- Adds unit + e2e tests
- Bumps version
- Writes PROGRESS entry
- Commits + tags + pushes
- Orchestrator publishes the Release

Bundle gaps into releases by theme (e.g. v0.2.8 = "Live mode parity",
v0.2.9 = "Export round-trip fidelity") — not one release per gap.

### Phase D — Sign-off audit

Final agent: run the full feature matrix again with fresh data, verify
every previously-flagged gap is closed or explicitly deferred to v0.3.
Then session B negotiates v0.3.0 scope with the user.

## 6. Known gaps already identified in prior sessions

Session A flagged these but did not address them:

1. **Live mode + table cell editing interaction**: Phase 3.1.1's
   contentEditable cells were designed for Document mode. Live mode
   behaviour with the new cell rendering is undocumented and untested.

2. **Source mode toolbar visibility**: toolbar only shows in
   `editMode === 'wysiwyg'`. Whether Source mode should expose a
   reduced toolbar (e.g. just "go back to Document") is a UX call.

3. **Mode switch + caret preservation**: switching mode currently keeps
   the doc but may not perfectly restore the caret/scroll position.
   No e2e covers this.

4. **Mermaid block in Document mode**: live preview renders the SVG.
   Editing the mermaid source requires going to active line. Is this
   the right pattern? Should there be a popover edit affordance
   similar to tables?

5. **Math block in Document mode**: same question as Mermaid.

6. **Front-matter editor**: currently a folded block decoration. No
   structured editor for the metadata fields (title, author, etc.).
   Is this a gap or a v0.3 feature?

7. **CriticMarkup display in cells**: Phase 3.1.2 explicitly left
   CriticMarkup literal inside cells. Cross-mode behaviour
   inconsistent (cells render `**bold**` but not `{++…++}`).

8. **Export round-trip**: a doc with all 5 CriticMarkup kinds +
   citations + math + memos + Mermaid + tables — does Pandoc DOCX
   round-trip preserve everything? Has not been verified.

9. **Image paste in non-saved docs**: image picker requires saved
   doc path. Behaviour when user pastes into an `untitled.md` is
   currently "alert + skip", but the UX could prompt to save first.

10. **Recent Folders menu**: separate from Recent Files but the menu
    structure / persistence has not been audited end-to-end.

11. **Korean IME + composition state across all decorations**:
    invariants #1, #6, #11, #12 each carve out an exception. The full
    matrix of "IME composition mid-X inside Y" has not been
    e2e-exhausted.

12. **Memo sidecar across mode switch**: the memo panel's "current
    paragraph" tracking — does it still work cleanly when the user
    flips between modes?

These are HINTS for the Phase A agent, not a complete list. The
agent should find more.
